# 외부 문서 인물·장소 후보 과다 추출

## 문제

외부 TXT 문서를 규칙 기반으로 분석하면 조사 앞의 모든 명사구를 인물로, 장소 접미사 앞의 문장 일부를 장소로 취급했다.

대표 오탐:

- 장소: `마나님을 전찻길`
- 장소: `마음은 급하고 불길`
- 인물: `모양`
- 인물: `조밥`

## 원인

- 인물 규칙이 품사나 행위 문맥을 구분하지 않고 반복되는 `명사+조사`를 모두 인물 후보로 승격했다.
- 장소 정규식이 공백을 허용한 상태로 탐욕적으로 일치해 장소 핵심 명사 앞의 형용사·부사·조사·다른 명사까지 하나의 이름에 포함했다.
- seed 후보 생성 뒤 동일한 광역 정규식을 다시 적용해 오탐 객체와 mention이 중복 생성됐다.
- Python/Ollama 전처리에도 같은 광역 정규식이 있어 잘못된 후보가 LLM 문맥으로 전달됐다.

## 해결

- 공백이 없는 어절 단위로 조사와 장소 접미사를 분리한다.
- 인물은 명시적 사람 지칭어이거나, 반복 출현하면서 주격·화제 조사와 사람 행위 문맥이 확인된 이름만 허용한다.
- 장소는 장소 핵심 명사만 저장하고 처소 조사, 이동 문맥, 장소성이 강한 합성어를 근거로 판정한다.
- `불길`, `징역`, `시집`, `가능성`, `들어가게`처럼 장소 접미사와 철자가 겹치는 일반어·활용형을 제외한다.
- seed 이후의 중복 광역 후보 패스를 제거하고 mention을 단어 경계 기준으로 생성한다.
- 브라우저 분석기, Python 형태소 fallback, Node 서버 fallback에 같은 보수적 경계를 적용한다.

## 회귀 기준

- `마나님은 전찻길을 건너갔다`에서 인물 `마나님`, 장소 `전찻길`을 각각 생성한다.
- `마음은 급하고 불길했다`에서 장소 `불길`을 생성하지 않는다.
- `모양`, `조밥`을 인물로 생성하지 않는다.
- 외부 문서로 불러온 「감자」에서 공백이 포함된 문장 조각을 장소명으로 생성하지 않는다.
